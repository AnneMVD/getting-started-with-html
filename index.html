The Basics of the Web and HTML

The Web is a collection of documents.
It's sort of like a library. It can keep lots of different things: text, pictures, videos, etc.
But the biggest part of the web is HTML.

HTML is a language that tells the computer how to show all the things on the web.
One way to think about it is to imagine HTML is a hi-tech display case.
It doesn't permanently change the thing you are showing, it just lets you look at it in a different way.
For example, look at these items in their display cases:

Imagine you could shine a light on the cases to emphasize the contents within.
You could slant the cases to the side so we'd have to look at them on an angle.
You could move the middle case to the end so it's in a different position, or move them away from each other so there are big spaces in between each case.
These hi-tech cases even have zoom lenses to make everything inside look bigger or smaller.
But remember, you can always take the item out of the case and it's back to its original state. Then you can put it in a new case to make it look different all over again!

These hi-tech display cases can even show things or bring things that are located in a different case somewhere in the world.
And most amazingly, certain display cases can even bring you to a different room of display cases if you touch it.
We call these special connections references and hyperlinks.

This is what HTML does to all the text, pictures, videos, etc on the web.
All of these things need display cases to show them correctly on the web, so as you can imagine that there is a lot of HTML on the web.

Communicating in HTML

Like all languages, HTML has a vocabulary. And HTML uses this vocabulary to tell the computer how to show things.
And for some things, we tell the computer to start showing something a certain way, but we also need to tell the computer to stop showing it a certain way.
So computers that communicate in HTML are a little "stupid". If we don't tell it to stop doing certain things, it will keep going like an Energizer bunny on a perpetual motion battery.
So we have to be sensitive to how the computer understands HTML. It requires a little different thinking than English or even talking to humans.

The vocabulary of HTML doesn't use words, it uses tags. These are sets of symbols that tell a computer to show something, for example, as
emphasized
bold
or even

spaced

apart.
When we combine these symbols with the thing we want to show, it''s an element
Elements can also be either inline or block.

Imagine writing a school essay or notes on a piece of ruled paper. We usualy write across the page, staying on one line until we run out of room.
Then we go to the next line and keep writing across until we run out of page.
Sometimes are sentences are short so we don't have to go all the way to the end of the page.
But we still start the next sentence on the next line at the beginning and go until we run out of room or don't need anyore room. Then we go to the next line.
HTML Inline Elements are similar becase they like to stick to the lines.
But Block Elements don't stick to the lines. They stick to boxes.

Think about this picture:

If you were taking notes using this paper, would you start writing info in the "cue" section and just write all the way across into the "lecture notes" section?
No, you would get to the line and move your pen back to the left edge of the page before writing any more. The lecture section is for different info.
So block elements are sort of like this, they stay in their boxes, even if we can't visibly see the boxes in our HTML or on the webpage.

HTML has a lot of tags and elements we can use so we can find a lot of different ways to show all the stuff on the web!
